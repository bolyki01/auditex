# Auditex Reviewer Handoff

This folder is a reviewer handoff pack for a senior engineer or architect taking over `Auditex`.

Read these files in order:

1. `REVIEW-INSTRUCTIONS.md`
2. `FEATURE-INVENTORY.md`
3. `VALIDATION-MATRIX.md`
4. `REFERENCE-MAP.md`
5. `KNOWN-GAPS-AND-IMPROVEMENTS.md`
6. `DELIVERABLE-CONTRACT.md`

Purpose:

- explain what the project is trying to be
- show what is already implemented
- show what still needs to be challenged or extended
- define how the reviewer should inspect it
- define the exact return artifact expected from the reviewer

Review stance:

- assume the current implementation is useful but incomplete
- assume the reviewer is expected to challenge architecture, correctness, scale, and product shape
- assume the reviewer is allowed to improve code, tests, docs, runtime design, and operator workflows
